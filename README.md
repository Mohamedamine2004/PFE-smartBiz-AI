## 🎯 Overview

**SmartBiz AI** is a full-stack SaaS platform designed specifically for **Small and Medium-sized Enterprises (SMEs)**. It solves three critical pain points that SME executives face daily:

| Problem | SmartBiz AI Solution |
|---|---|
| 📁 Financial data scattered across spreadsheets | Centralized import pipeline from Excel |
| 💰 No affordable business valuation tools | AI-assisted multi-method valuation engine |
| 🔮 No forward-looking analytics | ML-powered revenue & cash flow forecasting |

The platform is **multi-tenant**, meaning each company's data is fully isolated. A single instance serves multiple companies simultaneously, each with their own users, roles, and financial history.

## ✨ Key Features

### 🏦 Financial Data Management
- **Excel Import Pipeline** — Upload multi-sheet Excel files containing P&L statements, balance sheets, and strategic KPIs
- **Smart ETL Processing** — Automatic normalization, validation, and period-based indexing of financial metrics
- **Import History** — Full audit trail of every data ingestion with batch tracking

### 🤖 AI-Powered Business Valuation
- **Multiple Methods** — DCF (Discounted Cash Flow), EV/EBITDA multiples, Revenue multiples, Asset-based approaches
- **Gemini AI Integration** — Google's Gemini model explains every valuation with natural-language justifications
- **Valuation History** — Save, compare, and track valuations over time with labeled snapshots

### 📈 Predictive Analytics
- **CatBoost ML Models** — Trained on real SEC (U.S. Securities and Exchange Commission) financial filings
- **Multi-Horizon Forecasting** — 1-year, 2-year, and 3-year revenue/valuation predictions
- **Confidence Intervals** — Probabilistic outputs with upper/lower bounds
- **Anomaly Detection** — Alerts when financial metrics deviate from expected patterns

### 📄 Professional Report Generation
- **AI-Authored Reports** — Full executive summaries generated by LLM (Gemini / OpenRouter / Grok)
- **PDF Export** — Branded, multi-page PDF reports with charts and company logo
- **Excel Export** — Raw data and KPIs exported in structured spreadsheets
- **Multi-Language** — Reports generated in French, Arabic, or English
- **Report Queue** — Async report generation with real-time status tracking

### 👥 Team & Role Management
- **Multi-Tenancy** — Complete data isolation per company
- **Role-Based Access Control** — `OWNER`, `ADMIN`, `COLLAB`, `READER` roles with granular permissions
- **Email Invitations** — Invite teammates via email with token-based onboarding
- **Multi-Company Users** — A single user can belong to multiple companies with different roles in each

### 🔔 Notification System
- **In-App Notifications** — Real-time alerts for imports, valuations, predictions, and team events
- **Notification Types** — Import success, valuation complete, prediction ready, anomaly detected, team invites, system alerts
- **Read/Unread Tracking** — Notification center with mark-as-read functionality

### 🎨 Premium UI/UX
- **Dark / Light Mode** — System-aware theme with user preference persistence
- **Responsive Design** — Mobile-first layout with collapsible sidebar and touch-friendly targets
- **Skeleton Loaders** — Professional shimmer loading states instead of spinners
- **Empty States** — Helpful, action-oriented empty state illustrations
- **Keyboard Shortcuts** — Power-user shortcuts (`g+d`, `g+i`, `n+v`, `?` for help)
- **Toast Notifications** — Beautiful non-blocking feedback for all user actions
- **Framer Motion** — Smooth page transitions and micro-animations

### 🌍 Internationalization (i18n)

## 🏗️ Architecture

```
┌──────────────────────────────────────────────────────────┐
│                    SmartBiz AI Platform                    │
├──────────────┬───────────────────┬───────────────────────┤
│   Frontend   │      Backend      │      ML Engine        │
│  React 19    │    NestJS 11      │   FastAPI + Python    │
│  TypeScript  │    TypeScript     │   CatBoost / XGBoost  │
│  Vite 8      │    Prisma ORM     │   Pandas / NumPy      │
│  TailwindCSS │    PostgreSQL     │   SEC Financial Data  │
│  Recharts    │    JWT Auth       │                       │
│  Zustand     │    Swagger        │                       │
│  Framer      │    Pino Logger    │                       │
│  Motion      │    Sentry         │                       │
└──────┬───────┴────────┬──────────┴──────────┬────────────┘
       │                │                     │
       │   REST API     │   Internal API      │
       │◄──────────────►│◄───────────────────►│
       │                │                     │
       │         ┌──────▼──────┐              │
       │         │ PostgreSQL  │              │
       │         │  Database   │              │
       │         └─────────────┘              │
       │                                      │
       └──────────── Docker Network ──────────┘
```

**Communication Flow:**
1. User interacts with the **React frontend** (port `5173`)
2. Frontend calls the **NestJS REST API** (port `3001`)
3. Backend processes requests, persists via **Prisma → PostgreSQL**
4. For ML predictions, backend calls the **FastAPI ML Engine** (port `8000`)
5. ML Engine returns predictions/valuations back to the backend

## ⚡ Quick Start

### Prerequisites

Make sure you have the following installed:
- [Docker Desktop](https://www.docker.com/products/docker-desktop/) ≥ 4.x
- [Git](https://git-scm.com/)

### 1. Clone the Repository

```bash
git clone https://github.com/Mohamedamine2004/PFE-smartBiz-AI.git
cd PFE-smartBiz-AI
```

### 2. Configure Environment Variables

Copy the example environment files and fill in your secrets:

```bash
# Backend environment
copy apps\backend\.env.example apps\backend\.env
```

Open `apps/backend/.env` and configure:

```env
# Database (auto-configured in Docker, change for external DB)
DATABASE_URL=postgresql://smartbiz_user:smartbiz_password@localhost:5432/smartbiz_db

# JWT Secrets — CHANGE THESE in production!
JWT_ACCESS_SECRET=your-super-secret-jwt-access-key
JWT_REFRESH_SECRET=your-super-secret-jwt-refresh-key
JWT_EXPIRES_IN=15m
JWT_REFRESH_EXPIRES_IN=7d

# Google Gemini AI (for valuations & report generation)
GEMINI_API_KEY=your-gemini-api-key

# Alternative LLM: xAI Grok
XAI_API_KEY=your-xai-api-key

# Email (Gmail SMTP example)
MAIL_HOST=smtp.gmail.com
MAIL_PORT=587
MAIL_USER=your-email@gmail.com
MAIL_PASSWORD=your-app-password

# Sentry (optional, for error monitoring)
SENTRY_DSN=your-sentry-dsn
```

### 3. Start with Docker

**Windows (recommended):**
```bat
docker-start.bat
```

**Linux / macOS:**
```bash
docker compose up --build
```

### 4. Access the Application

| Service | URL | Description |
|---|---|---|
| 🖥️ Frontend | http://localhost:5173 | React application |
| ⚙️ Backend API | http://localhost:3001/api/v1 | NestJS REST API |
| 📚 Swagger Docs | http://localhost:3001/api/docs | Interactive API docs |
| 🤖 ML Engine | http://localhost:8000 | FastAPI prediction service |
| 🛢️ Prisma Studio | http://localhost:5555 | Database browser |

## 🤖 ML Engine Deep Dive

The ML Engine is the core competitive advantage of SmartBiz AI. It provides business valuation predictions based on real-world financial data from publicly listed companies.

### Training Data

- **Source**: SEC (U.S. Securities and Exchange Commission) EDGAR financial filings
- **Volume**: ~1.75M rows for Y1 predictions, scaled down for Y2/Y3
- **Features**: Revenue, EBITDA, total assets, equity, debt ratios, sector, market cap, and 20+ financial KPIs
- **Target**: Enterprise value at 1, 2, and 3 years forward

### Model Architecture

```
Input Features (Financial KPIs)
         │
         ▼
Feature Engineering Pipeline
  ├── Ratio calculation (D/E, P/E, EV/EBITDA…)
  ├── Clip bounds normalization (clip_bounds_Y*.json)
  └── Categorical encoding (sector, country)
         │
         ▼
CatBoost Gradient Boosting
  ├── model_Y1.cbm  → 1-year valuation
  ├── model_Y2.cbm  → 2-year valuation
  └── model_Y3.cbm  → 3-year valuation
         │
         ▼
Prediction Output
  ├── Enterprise Value estimate
  ├── Confidence interval (±%)
  └── Feature importance breakdown
```

### Training Pipeline

```bash
cd apps/ml-engine

# Step 1: Build feature engineering pipeline
python 1_build_pipeline.py

# Step 2: Train CatBoost models
python 2_train_model.py

# Step 3: Evaluate model performance
python 3_test_model.py

# Or run everything at once
python 4_run_all.py
```

## 📊 Data Model

The database is built on **PostgreSQL 16** and managed via **Prisma ORM**.

```
Company (multi-tenant root)
  ├── User[]             — Company members with roles
  ├── UserCompany[]      — Multi-company membership junction
  ├── ImportBatch[]      — Excel import sessions
  │     └── FinancialData[]  — Individual metric rows (metric, value, period)
  ├── Prediction[]       — ML prediction jobs
  ├── SavedValuation[]   — Saved valuation snapshots
  ├── Report[]           — Generated AI reports
  └── SuperAdmin?        — Platform administrator link

User
  ├── isEmailVerified    — Email verification workflow
  ├── inviteToken        — Team invitation token
  └── UserPreference     — Theme & language preferences

Notification            — Per-user notification inbox
InvitationRequest       — Landing page lead capture
```

**User Roles:**

| Role | Description |
|---|---|
| `OWNER` | Full control, created the company |
| `ADMIN` | Can manage users, data, and settings |
| `COLLAB` | Can import data and run analyses |
| `READER` | Read-only access to dashboards |

## 🔐 Authentication & Multi-Tenancy

SmartBiz AI uses a **stateless JWT authentication** system with refresh token rotation:

1. **Login** → Returns `access_token` (15 min) + `refresh_token` (7 days, httpOnly cookie)
2. **Token Refresh** → Silent re-authentication using the refresh cookie
3. **Email Verification** → Required before accessing the platform
4. **Password Reset** → Secure token-based flow via email
5. **Team Invitations** → Email links with `inviteToken` for new member onboarding

**Multi-tenancy** is enforced at the database level — every query is automatically scoped to `companyId` extracted from the JWT payload. A user cannot access data from another company even with a valid token.
